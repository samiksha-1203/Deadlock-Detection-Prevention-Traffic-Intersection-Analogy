Model Mapping:

  • üöó Cars = Processes (P1, P2, P3, P4)
  • üõ£Ô∏è Lanes = Resources (R_North, R_South, R_East, R_West) - Single instance each
  • ‚ö° Each car needs 2 resources: Entry lane + Exit lane to complete journey

Three Modes:

  • üî¥ Detection Mode: Allows all requests, then detects deadlock using RAG cycle detection
  • üü¢ Avoidance Mode (Banker's): Checks if granting request keeps system in safe state before allowing
  • üü° Prevention Mode: Uses resource ordering (N‚ÜíE‚ÜíS‚ÜíW) to prevent circular wait

Key Concepts:

  • Max Matrix: Declared upfront - each car declares its full resource need (entry + exit lane)
  • Safe State: A state where there exists a sequence to complete all processes
  • Resource Ordering: If all processes request resources in the same order, no circular wait can form
